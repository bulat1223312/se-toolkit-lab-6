# План реализации Task 2

## 1. Инструменты
- **read_file(path)** – читает содержимое файла. Проверка: `os.path.abspath(path).startswith(project_root)`.
- **list_files(path)** – возвращает список файлов и папок. Аналогичная проверка.

## 2. Схемы function calling
Опишем JSON-схемы для OpenAI API (или другого LLM) с параметрами.

## 3. Агентский цикл
- Инициализируем историю сообщений с вопросом пользователя.
- Цикл до 10 итераций:
  - Отправляем запрос LLM с доступными инструментами.
  - Если есть `tool_calls` – выполняем каждый, добавляем результат как сообщение с ролью `tool`, продолжаем.
  - Если нет `tool_calls` – финальный ответ, извлекаем `source`, выходим.
- Если превышен лимит – возвращаем последний ответ.

## 4. Извлечение source
В системном промпте просим LLM в конце ответа добавлять строку `Source: wiki/путь#якорь`. Регуляркой вытаскиваем её.

## 5. Тесты
Добавим два теста:
- `test_merge_conflict` – проверяет, что в `tool_calls` есть `read_file` с `wiki/git-workflow.md`, а `source` содержит `#resolving-merge-conflicts`.
- `test_list_wiki` – проверяет вызов `list_files` с путём `wiki`.
