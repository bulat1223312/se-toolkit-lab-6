# Task 3: The System Agent – план реализации

## Реализация query_api
- Функция будет выполнять HTTP-запросы к AGENT_API_BASE_URL с заголовком Authorization: Bearer {LMS_API_KEY}.
- Возвращает JSON со статусом и телом ответа.
- Схема для OpenAI function calling:
  - name: query_api
  - description: "Call the backend API to retrieve dynamic data or test endpoints."
  - parameters: method (string, enum GET/POST), path (string), body (string, optional)

## Обновление системного промпта
Добавить инструкцию:
- Используй read_file/list_files для вопросов о коде, конфигах, вики.
- Используй query_api для вопросов о данных (сколько элементов, код ответа без авторизации) и для выявления ошибок API.
- Если нужно диагностировать ошибку, вызови query_api, затем прочитай соответствующий файл с кодом.
